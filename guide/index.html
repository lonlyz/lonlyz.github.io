# 设计模式

设计模式代表了最佳的实践，通常是开发过程中面临问题的一般解决方案，是经过长时间试验和错误中总结出来的。

设计模式的出现将软件设计的六大原则完美的体现出来。

【六大原则】

  • 单一职责原则

    一个类只负责一件事情

  • 里氏替换原则

    子类可以替换父类并可以正常工作，所有引用基类的地方必须能透明的使用子类。

  • 接口隔离原则

    一个接口拥有的行为应尽可能的小，ps：跟单一职责原则有点类似

  • 依赖倒置原则

    高层模块不应该依赖于低层模块，二者都应依赖于抽象，抽象不应该依赖于细节，细节应该依赖于抽象

  • 迪米特原则

    一个类不应该和其它类存在过多的耦合

  • 开闭原则

    对修改关闭，对扩展开放

用抽象构建框架，用细节实现扩展
